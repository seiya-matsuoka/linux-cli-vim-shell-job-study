# Phase 1 Process / Job Intro

## 目的

- プロセスとジョブの入口を理解する
- `jobs`, `fg`, `bg`, `kill` の基本を説明できるようにする
- 今後の Shell Script、cron、systemd timer 学習の土台を作る

## 学習項目

- `ps`
- `top`
- `&`
- `jobs`
- `fg`
- `bg`
- `kill`

## 用語整理

### process

実行中のプログラムそのもの。  
Linux では、実行中のコマンドやアプリケーションはプロセスとして管理される。

### job

現在のシェルから起動した処理のまとまり。  
Shell 上で `&` を付けて起動した処理や、`Ctrl + Z` で停止した処理はジョブとして扱える。

### foreground

現在のターミナルを占有して実行されている状態。  
ユーザー入力を直接受け取る。

### background

ターミナルの裏で実行されている状態。  
シェルに制御が戻るので、別のコマンドを続けて打てる。

### PID

Process ID。  
各プロセスに付与される識別番号。

### signal

プロセスへ送る通知。  
たとえば終了依頼や停止依頼などがある。

## 学んだコマンド

### `ps`

現在のプロセス状況を表示する。

例:

```bash
ps
ps -ef
ps -ef | grep bash
```

### `top`

リアルタイムでプロセスや CPU / メモリ使用状況を見る。

例:

```bash
top
```

終了は `q`。

### `&`

コマンドをバックグラウンド実行する。

例:

```bash
sleep 300 &
```

### `jobs`

現在のシェルで管理しているジョブを表示する。

例:

```bash
jobs
jobs -l
```

### `fg`

バックグラウンドまたは停止中のジョブを foreground に戻す。

例:

```bash
fg %1
```

### `bg`

停止中のジョブをバックグラウンドで再開する。

例:

```bash
bg %1
```

### `kill`

ジョブやプロセスへ終了シグナルを送る。

例:

```bash
kill %1
kill 12345
```

## 実際にやったこと

### `ps` でプロセス確認

以下を実行した。

```bash
ps
ps -ef | head
ps -ef | grep bash
```

この学習で分かったこと。

- `ps` は現在のシェル周辺のプロセス確認に使える
- `ps -ef` はより広い範囲のプロセス一覧を確認できる
- `grep` と組み合わせると目的のプロセスを探しやすい

### `top` で状況確認

以下を実行した。

```bash
top
```

この学習で分かったこと。

- CPU 使用率、メモリ使用量、ロードアベレージなどをリアルタイムで見られる
- 実務で「重い」「何か動いているか確認したい」ときの入口になる
- まずは詳細に読むより、見たことがある状態にしておくことが重要

### バックグラウンド実行

以下を実行した。

```bash
sleep 300 &
jobs
ps -ef | grep sleep
```

この学習で分かったこと。

- `&` を付けるとコマンドをバックグラウンド実行できる
- `jobs` は現在のシェル起点のジョブを見るコマンド
- `ps` はシステム上のプロセスを見るコマンド
- `jobs` と `ps` は似ているが視点が違う

### 停止と再開

以下を実行した。

```bash
sleep 300 &
jobs
fg %1
```

foreground に戻した後、`Ctrl + Z` で停止した。
その後、以下を実行した。

```bash
jobs
bg %1
jobs
```

この学習で分かったこと。

- `fg` はジョブを前面に戻す
- `Ctrl + Z` は終了ではなく一時停止
- `bg` は停止したジョブをバックグラウンドで再開する
- ジョブ制御はシェルを理解する上で重要な基礎

### `kill`

以下を実行した。

```bash
jobs -l
kill %1
```

必要に応じて PID 指定も確認した。

```bash
ps -ef | grep sleep
kill <PID>
```

この学習で分かったこと。

- `kill` はジョブ番号でも PID でも使える
- `kill` は「強制終了コマンド」というより、「シグナルを送る」コマンドとして理解した方が正確
- 今回は基本として終了依頼を送る使い方を確認した

## `Ctrl + Z` の意味

`Ctrl + Z` は、foreground で動いているジョブを停止状態にする操作。  
削除や終了ではなく、一時停止である点が重要。

停止後は `jobs` で状態確認できる。  
必要に応じて `bg` で再開したり、`fg` で前面へ戻したりできる。

## `jobs`, `fg`, `bg`, `kill` の関係

基本の流れは以下。

1. `sleep 300 &` でバックグラウンド起動
2. `jobs` で確認
3. `fg %1` で foreground に戻す
4. `Ctrl + Z` で停止
5. `jobs` で停止状態確認
6. `bg %1` でバックグラウンド再開
7. `kill %1` で終了

この一連の流れで、ジョブ制御の最初の全体像を確認できた。

## `jobs` と `ps` の違い

### `jobs`

- 現在のシェルで起動・管理しているジョブを見る
- シェル起点の視点

### `ps`

- システム上のプロセスを見る
- OS 全体の視点

つまり、`jobs` は Shell のジョブ制御、`ps` はプロセス確認、と整理できる。

## メモ

- `ps` はプロセス確認の基本
- `top` は状況観測の入口
- `&` はバックグラウンド実行
- `jobs` は現在のシェルのジョブ一覧
- `fg` は foreground へ戻す
- `bg` は停止ジョブを background で再開する
- `kill` はジョブ / プロセスへシグナルを送る
- この内容は今後の Shell Script、cron、systemd timer 学習につながる
